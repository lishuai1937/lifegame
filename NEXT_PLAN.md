# LifeGame 后续开发方案
# 日期: 2026-04-11

## 当前状态

已完成:
- 2D 棋盘 + 3D 开放世界核心循环跑通
- 摇骰子 → 走格子（动画） → 进入 3D 世界 → ESC 暂停/退出 → 回棋盘
- 100 个年龄格子，每格有事件数据（JSON 驱动）
- SceneGenerator 根据 SceneId 生成不同场景（家、学校、城市、医院等）
- NPC 系统（影子人、揭示、对话、社交关系、手机系统）
- 隐藏 Karma 系统、梦想/职业系统、世界事件系统
- 像素风 UI 框架（主菜单、棋盘 HUD、事件弹窗、死亡画面、暂停菜单）
- 52 个 NPC 对话 JSON 文件

存在的问题:
- 3D 世界全是 primitive 方块，没有美术资源
- NPC 交互只有 Debug 日志，没有对话 UI 显示
- 人物模型是胶囊体，没有动画
- UI 风格统一但缺乏精细度
- 很多系统写好了但 UI 没接上（梦想选择、NPC 邀请、手机、日记等）

---

## 阶段一: 核心交互补全（优先级最高）

目标: 让玩家能真正跟世界互动，而不只是看 Debug 日志

### 1.1 对话 UI 接入
- DialogueUI.cs 已经写好，但没被 Builder 创建到场景里
- 需要在 BuildUICanvas 里创建对话面板（说话人名字、对话文本、选项按钮）
- 进入 3D 世界后，靠近 NPC 按 E 弹出对话，选择选项影响 Karma
- 对话结束后 NPC 从影子变成真人（揭示动画）

### 1.2 梦想/职业选择 UI
- 在年龄 6/18/22/30 触发梦想选择界面
- 22 个职业选项，每个有描述和 Buff 说明
- 选择后影响后续游戏（金币倍率、死亡概率、社交能量等）

### 1.3 NPC 邀请 UI
- 棋盘上推进年龄时，亲密 NPC 会发来邀请（婚礼、生日、危机求助）
- 弹窗显示邀请内容，玩家选择参加/拒绝
- 参加消耗能量但提升关系，拒绝降低关系

### 1.4 手机 UI
- 12 岁解锁手机，18 岁升级智能手机
- 联系人列表（按亲密度排序）
- 打电话、发短信、借钱、还钱、邀请功能

## 阶段二: 美术资源替换

目标: 把 primitive 方块替换成像样的视觉效果

### 2.1 2D 棋盘美术
- 格子贴图: 每个年龄阶段不同风格的平台 Sprite
  - 童年(1-12): 彩色积木风
  - 青春(13-17): 校园绿色调
  - 青年(18-30): 都市金色调
  - 壮年(31-50): 灰白商务风
  - 中年(51-65): 暖棕色调
  - 老年(66-100): 深灰淡紫
- 玩家角色: 像素小人 Sprite（带行走帧动画）
- 背景: 分层视差滚动背景（星空/云层/远景建筑）
- 骰子: 像素骰子动画

### 2.2 3D 世界美术
- 推荐工具:
  - Meshy / Tripo3D: 文字生成 3D 模型（建筑、道具、家具）
  - Mixamo: 免费角色动画（走路、跑步、交互）
  - Stable Diffusion + ComfyUI: 生成贴图、UI 素材、CG 插图
- 场景模板（6 套基础场景，通过参数变化复用）:
  - 家（婴儿房/儿童房/青少年房/父母家/老年公寓）
  - 学校（幼儿园/小学/中学/高中/大学）
  - 城市（办公室/创业公司/公寓/郊区/公园）
  - 医院
  - 乡村
  - 海滩/旅行
- NPC 模型: 低多边形风格，影子状态用全黑 Shader

### 2.3 UI 美术
- 像素字体: Press Start 2P 或类似中文像素字体
- 按钮贴图: 像素风格按钮（正常/悬停/按下三态）
- 面板背景: 带边框的像素风面板
- 图标: 金币、心形（Karma）、手机、骰子等像素图标

## 阶段三: 系统 UI 接入

目标: 把已经写好的后端系统接上前端 UI

### 3.1 玩家状态面板（详细版）
- 按 Tab 打开完整状态面板
- 显示: 年龄、金币、职业、家庭背景、资产列表
- 隐藏属性不显示数值，用描述文字（"你感觉自己很有魅力"）

### 3.2 日记系统 UI
- 自动记录人生大事
- 可以在棋盘上按 J 打开日记回顾
- 死亡画面展示日记精华

### 3.3 遗愿系统 UI
- 70 岁解锁
- 弹窗让玩家写下 3 个遗愿
- 死亡时检查是否完成，影响结局文字

### 3.4 速度选择优化
- 每 20 格弹一次速度选择
- 加入更多信息: 当前年龄、已走格数、剩余格数
